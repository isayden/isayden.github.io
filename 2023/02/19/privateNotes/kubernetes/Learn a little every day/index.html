用于查看和修改内核参数。其中，net.ipv4.ip_local_port_range 是一个 TCP/IP 协议栈相关的参数，用于指定本地端口范围，即本地分配的端口号的范围。

当一个进程通过 socket 调用向外发起 TCP 连接时，内核会从这个范围内随机分配一个本地端口号。通常情况下，该参数的默认值为 32768 ~ 60999。

这条命令的作用是查看当前系统的 net.ipv4.ip_local_port_range 参数值，以确定本地端口号的范围。当出现 Kubernetes NodePort 与本地端口范围冲突的问题时，可以通过这个命令来检查当前系统的本地端口范围，并根据需要修改这个参数的值，以避免冲突。

用于获取指定集群中所有服务的 NodePort 端口号，并使用 jsonpath 格式化输出结果。

  • kubectl: Kubernetes 命令行工具。
  • get svc: 获取 Kubernetes 集群中的所有服务（Service）对象。
  • --all-namespaces: 在所有命名空间中查询服务对象。
  • -o=jsonpath='{.items[*].spec.ports[*].nodePort}': 将结果以 JSON 格式输出，并使用 jsonpath 过滤器来提取所有服务对象中的 nodePort 字段值。

其中，{.items[*].spec.ports[*].nodePort} 表示获取所有服务对象的 spec.ports 数组中的 nodePort 字段的值，多个值之间以空格分隔。

使用这个命令可以快速查看当前 Kubernetes 集群中所有服务的 NodePort 端口号，以帮助您检查端口是否与本地端口范围冲突。如果存在冲突，您可以根据需要修改本地端口范围或调整服务的 NodePort 端口号，以避免冲突。
